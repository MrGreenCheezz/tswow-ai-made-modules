# Переносимость модулей TSWoW

Этот файл описывает перенос исходных модулей в совместимую установку TSWoW
WotLK. `custom-npcbots` сюда не входит: NPCBots требует отдельной модификации
TrinityCore. Каталог `default` также не является исходным модулем — это
сгенерированный dataset текущей установки.

## Чистый TSWoW-комплект

В комплект без внешних runtime-компонентов входят:

- `attributes`
- `echoes`
- `base-building`
- `craft-all`
- `custom-companions`
- `custom-stats`
- `gem-abilities`
- `retail-talents`
- `simple-button-addon`
- `survival`
- `tswow-store`

Эти каталоги нужно целиком положить в `modules`, включить в один dataset и
запустить `build all`. Внутри игрового комплекта есть намеренные связи:

- `base-building` использует `echoes`, `craft-all`, `custom-stats`,
  `gem-abilities` и UI-ресурсы `tswow-store`;
- `custom-companions` использует `base-building`, `gem-abilities`,
  `retail-talents` и UI-ресурсы `tswow-store`;
- `gem-abilities` использует профессию лесозаготовки из `base-building`;
- `survival` использует укрытия и очаги из `base-building`;
- `retail-talents` использует UI-ресурсы `tswow-store`.

Поэтому игровой кластер переносится как единый набор. `build data` создаёт
dataset-specific каталоги рецептов, камней и талантов до компиляции
livescripts; закоммиченные заглушки не заменяют этот шаг. `build all` выполняет
нужные стадии в правильном порядке.

Клиентские аддоны используют штатные custom packets TSWoW. В целевом dataset
должен быть включён стандартный патч `client-extensions` (например,
`Client.Patches = ["all"]`). Стандартная конфигурация TSWoW может исключать его
по умолчанию; модульный API не умеет менять настройку dataset. Никакие файлы
автопатчера, ручной SQL-импорт или проектные правки TrinityCore комплекту не
нужны.

`tswow-store` сам создаёт схему магазина. Пустой каталог после сборки корректен:
товары и баланс пожертвований являются эксплуатационными данными сервера.

Каталог PatchForBuildings сохраняет 1569 исходных `gameobject_template.entry`
из bundled SQL (диапазоны `500000…539743` и `4000000…4002169`). На чистом или
текущем dataset они свободны; сторонний мод, уже занявший один из этих точных
ID, нужно согласовать до объединения наборов. Модуль больше не импортирует
остальные ~27,7 тыс. неиспользуемых шаблонов.

## Отдельные исключения

`modern-models` не входит в чистый комплект: Winged Lion хранится как
современный MD20 v274, который клиент 3.3.5a не читает. Для него требуется
внешний WarcraftXL runtime; одного `build` недостаточно.

`simple-button-addon` — самостоятельное демонстрационное окно и не требуется
системе `retail-talents`.

## Что считать голым модулем

Для распространения нужны исходные endpoint-каталоги (`addon`, `assets`,
`datascripts`, `livescripts`, `shared`), `.conf`, README и проверенные
generated-контракты `.ts`. Каталоги `build`, `log.txt`, `global.d.ts` и временные
`tsconfig.json` создаются TSWoW заново и в архив исходников не нужны.

Конвертеры, документация происхождения ресурсов и прочие authoring-инструменты
должны лежать в `tools` или в корне модуля, а не в `assets`: TSWoW зеркалирует
неизвестные расширения из `assets` в клиентский пакет. Общий portability-тест
проверяет, что `.exe`, скрипты инструментов и Markdown туда не возвращаются.
